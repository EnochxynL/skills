### 环境使用原则

* 用人为配置文件托管人为操作，与系统配置文件托管的自动操作分离开
* 所有 apt 和 dpkg 安装都使用 nala 安装取而代之，便于追溯记录

### 代码生成契约

当你的任务是生成或重构代码时，必须严格遵守以下架构规则（以 Python 为例）：

* **面向对象。**

1. **无全局变量**，一切逻辑封装为模块化的类。
2. **函数**：无状态的同步运算，仅通过 argument 和 return 交换数据。
3. **仿函数/类**：有状态记忆的同步类，内部属性作为状态寄存器，单功能调用优先取名 `__call__` 或 `run` 等。
4. **输入输出契约**：
   * argument → | MODULE | → return （管道，无记忆）
   * parameter → | MODULE | → signal （邮箱，有记忆）
   * 工具函数严格使用 argument/return；执行函数使用 parameter/signal。
5. **模块并行**：硬件时序要求弱的多模块可使用协程/线程并发。
6. **禁止自由 while/for**，每个线程/协程顶层最多一个主循环，内部状态切换采用三段式/四段式状态机，但允许临时的小循环。
7. **协程使用**：仅在函数需要等待非阻塞操作时才使用 async 函数。
8. **线程通信**：对于跨线程共享的可变状态，必须使用锁或消息队列。
9. **单一职责**：DAO、Processor、Controller 分离；数据与逻辑分离；前端与后端分离……
